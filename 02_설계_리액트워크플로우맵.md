# 02_설계_리액트워크플로우맵

## 환경
- 로컬: Node.js v24.14.1 / npm 11.11.0 (확인 완료)
- GitHub 저장소: 신규 생성 필요 (GitHub Pages 배포용)
- Gist PAT: 기존 토큰 재사용

## 기술 스택
- React + Vite (빠른 빌드, GitHub Pages 배포와 궁합 좋음)
- React Flow (트리+연결 혼합 구조, 드릴다운 네비게이션 구현)
- 순수 CSS (별도 UI 라이브러리 없이 최소 구성)
- 배포: GitHub Actions → GitHub Pages (SEED/VR대시보드와 동일 패턴)

## 데이터 흐름
1. 앱 최초 로드 → localStorage에 저장된 GitHub PAT 확인 (없으면 입력 요청)
2. PAT로 지정된 Gist ID의 JSON 파일을 GET → `{ nodes: [...], edges: [...] }` 형태로 파싱
3. React Flow 상태(mapStore)에 로드. HQ 노드부터 방사형 배치 렌더링
4. 사용자가 브랜치/유닛 클릭 → mapStore의 "현재 경로(path)" 갱신 → 해당 노드의 자식들만 방사형 재배치
5. 자식이 없는 노드 클릭 → DetailPanel 표시 (텍스트/링크)
6. 사용자가 +버튼으로 노드 추가/삭제, 또는 두 노드를 선택해 연결선(방향+라벨) 생성 → mapStore 갱신 → Gist PATCH로 즉시 반영(또는 "저장" 버튼으로 일괄 반영 — 구현 단계에서 확정)
7. 채팅(Claude)에서 구조 편집 요청 시 → Claude가 동일 Gist JSON을 직접 GET/PATCH (앱과 같은 데이터 소스 공유)

## 설계 범위 밖 (구현 단계에서 결정)
- 노드/엣지 CRUD 시 즉시저장 vs 수동저장 버튼
- 방사형 배치 애니메이션 세부 값
- 모바일 제스처 세부 (핀치줌 vs 버튼)
