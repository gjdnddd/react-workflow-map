# 계획서_리액트워크플로우맵

## 상태
- 단계: 1단계(구상) 완료 대기 — 마스터 컨펌 필요

## 핵심 목적
HQ에서 시작하는 방사형 탐색 UI로 BRANCH → 유닛(무한 depth)을 드릴다운하며, 내가 운영하는 시스템·프로젝트들의 구조와 연관 관계를 직접 편집하고 탐색하는 인터랙티브 맵.

## 주요 사용 시나리오
1. **탐색**: HQ 화면에서 BRANCH들이 방사형으로 배치 → 브랜치 클릭 시 그 브랜치 중심으로 하위 유닛이 펼쳐짐 → 유닛 클릭 시 또 하위 유닛으로 드릴다운(무한 단계) → 더 이상 하위가 없는 최하위 노드는 그 자리에 상세 패널(텍스트/링크) 표시. 뒤로/축소 버튼으로 상위 레벨 복귀.
2. **편집**: +버튼으로 노드(브랜치/유닛) 직접 추가·삭제. 노드 간 연결선도 직접 그릴 수 있음(방향 화살표 + 관계 라벨, 예: '데이터 제공', '의존'). 노드 입력 필드는 자유 형식(구조는 설계 단계에서 확정).
3. **AI 보조 편집**: 구조가 복잡해지면 채팅(Claude)에게 "이 유닛들 연결해줘" 등으로 부탁 → Claude가 Gist JSON을 직접 읽고 편집. 앱 자체에는 AI 자동 분석 기능 없음.
4. **동기화**: 데이터는 외부 Gist(JSON)에 저장. 앱에서 GitHub 토큰을 입력(localStorage 저장)해 직접 Gist를 읽고 씀 — 기존 SEED/VR대시보드/mmNoti와 동일한 패턴.

## 기술 방향
- React Flow 기반
- 계층(트리, 무한 depth) + 브랜치/유닛 간 자유 연결선 혼합 구조
- GitHub Pages 정적 배포, 데이터는 Gist(JSON) 연동, GitHub 토큰은 localStorage
- 모바일 터치 지원 필수
- 추후 HQ 통합 대시보드에 탭 하나로 임베드 예정 (독립 실행도 가능해야 함)

## 초기 데이터
- 빈 상태로 시작. HQ 노드만 존재, 브랜치부터 사용자가 직접 채움.

## 가장 중요한 기능 (우선순위 1)
방사형 탐색 UI (HQ → 방사형 배치 → 클릭 드릴다운). 편집 기능은 그 다음 우선순위.

## 이번 버전 제외 범위 (후속 버전으로 미룸)
- 검색/필터
- AI 자동 구조 분석 / 자동 연결 추천
- 다중 사용자·권한 설정 등 부가 기능 전부

## 완료 기준
- HQ→BRANCH→유닛 무한 드릴다운 탐색이 동작
- +버튼 노드 추가/삭제, 방향+라벨 연결선 수동 생성이 동작
- Gist 읽기/쓰기(토큰 인증)로 데이터 영속화
- 모바일 터치로 동일 조작 가능
- GitHub Pages 배포 및 정상 작동 확인
