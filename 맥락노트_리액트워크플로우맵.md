## 트러블슈팅 기록 (2026-07-11)
- React Flow(`@xyflow/react`)의 `fitView`와 내장 엣지 렌더링이 이 자동화 브라우저 환경에서 동작하지 않음
  → 원인: ResizeObserver 기반 노드 측정(`node.measured`)이 이 환경에서 발화하지 않아, 뷰포트 맞춤과 엣지 경로 계산이 모두 실패
  → 조치: 뷰포트는 컨테이너 크기+노드 위치로 직접 계산해 `setViewport()` 호출. 엣지는 React Flow 내장 렌더링 대신 직접 계산한 좌표로 SVG 오버레이(`EdgeOverlay`)를 그려 우회
  → 실제 사용자 브라우저에서는 이 이슈가 재현되지 않을 가능성이 높음(자동화 환경 특유의 문제로 추정)이지만, 어차피 더 안정적인 방식이라 그대로 유지
- Gist 쓰기(PATCH)는 GitHub PAT이 필요해 AI가 직접 테스트 불가 (토큰 입력은 보안 규칙상 AI 금지 행동) → 사용자가 앱에서 "🔑 토큰" 버튼으로 직접 입력 후 저장 테스트 필요
- (2026-07-11 후속) 사용자가 실사용 중 "삭제 시 Gist 저장 실패, 새로고침하면 반영됨" 증상 보고
  → 원인: `deleteNode`가 `setEdges`를 `setNodes` 업데이터 콜백 안에 중첩 호출 → React StrictMode가 이 콜백을 의도적으로 이중 실행 → 저장 API가 두 번 호출되어 하나는 에러로 보이고 다른 하나는 성공해 반영됨
  → 조치: addNode/deleteNode/addEdge/deleteEdge 전부 클로저 state를 직접 사용하는 단일 setState 패턴으로 통일 (더미 토큰으로 PATCH 호출 횟수 계측해 1회로 줄어든 것 확인). GitHub에 커밋 d7003d4로 반영 및 push 완료
